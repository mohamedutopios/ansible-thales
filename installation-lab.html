1. Prérequis

Avant de commencer, assurez-vous d'avoir installé :

  1. Docker Desktop (avec l'intégration WSL si vous êtes sous Windows 11).
  2. Vagrant pour gérer les conteneurs Docker.
  3. WSL2 ou un environnement Linux fonctionnel.
  4. SSH installé sur votre hôte (Windows/Linux/Mac) pour tester la connectivité.

2. Structure de votre projet

Créez un répertoire pour votre lab :

mkdir ansible-lab
cd ansible-lab

3. Créer le fichier Vagrantfile

Voici un fichier Vagrantfile pour configurer une machine Ansible et deux nœuds administrés via Docker.

Vagrant.configure("2") do |config|
  # Machine Ansible
  config.vm.define "ansible" do |ansible|
    ansible.vm.provider "docker" do |docker|
      docker.image = "ubuntu:20.04"
      docker.cmd = ["tail", "-f", "/dev/null"]
      docker.ports = ["2222:22"] # Expose le port SSH pour Ansible
    end
    ansible.vm.hostname = "ansible"
    ansible.vm.provision "shell", inline: <<-SHELL
      apt-get update
      apt-get install -y ansible openssh-client openssh-server
      mkdir -p /var/run/sshd
      echo 'root:root' | chpasswd
      sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
      service ssh restart
    SHELL
  end

  # Node 1
  config.vm.define "node1" do |node1|
    node1.vm.provider "docker" do |docker|
      docker.image = "ubuntu:20.04"
      docker.cmd = ["tail", "-f", "/dev/null"]
      docker.ports = ["2223:22"] # Expose le port SSH pour Node1
    end
    node1.vm.hostname = "node1"
    node1.vm.provision "shell", inline: <<-SHELL
      apt-get update
      apt-get install -y openssh-server
      mkdir -p /var/run/sshd
      echo 'root:root' | chpasswd
      sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
      sed -i 's/#PasswordAuthentication no/PasswordAuthentication yes/' /etc/ssh/sshd_config
      service ssh restart
    SHELL
  end

  # Node 2
  config.vm.define "node2" do |node2|
    node2.vm.provider "docker" do |docker|
      docker.image = "ubuntu:20.04"
      docker.cmd = ["tail", "-f", "/dev/null"]
      docker.ports = ["2224:22"] # Expose le port SSH pour Node2
    end
    node2.vm.hostname = "node2"
    node2.vm.provision "shell", inline: <<-SHELL
      apt-get update
      apt-get install -y openssh-server
      mkdir -p /var/run/sshd
      echo 'root:root' | chpasswd
      sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
      sed -i 's/#PasswordAuthentication no/PasswordAuthentication yes/' /etc/ssh/sshd_config
      service ssh restart
    SHELL
  end
end

4. Démarrer le lab

  1. Lancez les machines avec Vagrant :

    vagrant up

  2. Vérifiez que les conteneurs sont actifs :

    docker ps

5. Configurer la machine Ansible

  1. Connectez-vous à la machine Ansible :

    vagrant ssh ansible

  2. Générez une paire de clés SSH sur Ansible :

    ssh-keygen -t rsa -b 2048 -f ~/.ssh/id_rsa -N ""

  3. Copiez la clé publique sur les nœuds :
    Pour Node1 :

    sshpass -p "root" ssh-copy-id -o StrictHostKeyChecking=no root@192.168.50.11

    Pour Node2 :

    sshpass -p "root" ssh-copy-id -o StrictHostKeyChecking=no root@192.168.50.12

  4. Testez la connexion sans mot de passe :

    • Pour Node1 :
      ssh root@192.168.50.11
    • Pour Node2 :
      ssh root@192.168.50.12

    Si cela fonctionne, passez à la configuration d'Ansible.

6. Configurer Ansible

  1. Créez un fichier d'inventaire :
    Sur la machine Ansible, créez un fichier appelé inventory :

    nano ~/inventory

    Ajoutez-y les informations sur vos nœuds :

    [nodes]
node1 ansible_host=192.168.50.11 ansible_user=root
node2 ansible_host=192.168.50.12 ansible_user=root

  2. Testez la connectivité avec Ansible :
    Exécutez la commande suivante :

    ansible -i ~/inventory nodes -m ping

    Si tout est configuré correctement, vous verrez une réponse similaire pour chaque nœud :

    node1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
node2 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}

7. Étapes supplémentaires (facultatives)

a. Automatiser l'installation d'un logiciel

Vous pouvez créer un fichier setup.yml pour installer un logiciel sur vos nœuds :

---
- name: Install Apache on nodes
  hosts: nodes
  become: yes
  tasks:
    - name: Ensure Apache is installed
      apt:
        name: apache2
        state: present

Exécutez-le avec :

ansible-playbook -i ~/inventory setup.yml

b. Ajouter d'autres nœuds

Pour ajouter un autre nœud, ajoutez simplement une définition dans le Vagrantfile et dans le fichier d'inventaire.
