Le fichier ansible.cfg est un fichier de configuration central d'Ansible qui permet de personnaliser le comportement global d'Ansible, y compris la gestion des connexions, des journaux, des modules, des rôles, et bien d'autres aspects. Il peut être utilisé pour adapter Ansible à vos besoins spécifiques et se trouve généralement dans un des emplacements suivants, en fonction de l'ordre de priorité :

  1. Dans le répertoire courant (local au projet) : Si un fichier ansible.cfg est présent dans le répertoire courant, il sera utilisé.
  2. Dans le répertoire utilisateur : ~/.ansible.cfg.
  3. Dans /etc/ansible/ansible.cfg : Le fichier de configuration global installé avec Ansible.

Structure et Sections de ansible.cfg

Le fichier est divisé en plusieurs sections, chacune définissant des paramètres spécifiques. Voici les sections principales et ce qu'elles peuvent contenir :

1. [defaults]

Cette section contient les paramètres par défaut pour la plupart des opérations Ansible.

Exemples de paramètres :

  • inventory : Chemin vers le fichier ou script d'inventaire à utiliser.
    inventory = ./inventory
  • remote_user : Utilisateur distant par défaut pour SSH.
    remote_user = ansible
  • timeout : Durée maximale en secondes pour établir une connexion.
    timeout = 30
  • forks : Nombre maximum de tâches simultanées.
    forks = 5
  • retry_files_enabled : Activer ou désactiver la génération de fichiers .retry.
    retry_files_enabled = False
  • host_key_checking : Désactiver la vérification de clé SSH.
    host_key_checking = False
  • stdout_callback : Définir le format de sortie des résultats (par exemple, default, yaml, json).
    stdout_callback = yaml

2. [privilege_escalation]

Cette section configure les paramètres pour l'élévation de privilèges (sudo).

Exemples de paramètres :

  • become : Activer ou désactiver become par défaut.
    become = True
  • become_user : Utilisateur cible pour l'élévation de privilèges.
    become_user = root
  • become_method : Méthode pour élever les privilèges (par exemple, sudo, su, pbrun, etc.).
    become_method = sudo
  • become_ask_pass : Demander le mot de passe pour l'élévation.
    become_ask_pass = False

3. [ssh_connection]

Cette section contrôle les paramètres pour les connexions SSH.

Exemples de paramètres :

  • ssh_args : Arguments personnalisés pour SSH.
    ssh_args = -o ControlMaster=auto -o ControlPersist=60s
  • pipelining : Activer le pipelining SSH pour améliorer les performances.
    pipelining = True
  • control_path : Chemin pour le fichier de contrôle SSH.
    control_path = %(directory)s/%%h-%%r
  • scp_if_ssh : Utiliser scp si sftp n'est pas disponible.
    scp_if_ssh = True

4. [inventory]

Cette section configure les paramètres liés aux inventaires.

Exemples de paramètres :

  • enable_plugins : Activer les plugins d'inventaire spécifiques.
    enable_plugins = host_list, script, yaml, ini, auto
  • cache : Activer la mise en cache des inventaires.
    cache = True
  • cache_plugin : Définir le plugin de cache (par exemple, JSON, Redis).
    cache_plugin = jsonfile
  • cache_timeout : Durée en secondes pour invalider le cache.
    cache_timeout = 300

5. [logging]

Cette section gère les options de journalisation.

Exemples de paramètres :

  • log_path : Chemin pour le fichier de journalisation.
    log_path = /var/log/ansible.log
  • log_level : Niveau de journalisation (DEBUG, INFO, WARNING, ERROR).
    log_level = DEBUG

6. [galaxy]

Cette section configure les options liées à Ansible Galaxy.

Exemples de paramètres :

  • server_list : Liste des serveurs Galaxy à utiliser.
    server_list = galaxy
  • role_skeleton : Chemin vers un squelette de rôle personnalisé.
    role_skeleton = /path/to/custom_skeleton
  • role_skeleton_ignore : Liste des fichiers/dossiers à ignorer lors de la création d’un rôle.
    role_skeleton_ignore = .git,.github

7. [callback]

Configure les options pour les plugins de callback.

Exemples de paramètres :

  • enabled : Activer les plugins de callback spécifiques.
    enabled = timer, profile_tasks
  • whitelist : Liste blanche des plugins de callback autorisés.
    whitelist = yaml, json, minimal

8. [retry]

Configure les paramètres pour les fichiers .retry.

Exemples de paramètres :

  • retry_files_enabled : Activer ou désactiver la création des fichiers de reprise.
    retry_files_enabled = False
  • retry_files_save_path : Chemin pour enregistrer les fichiers .retry.
    retry_files_save_path = /tmp/ansible-retries

9. [jinja2]

Personnalise les paramètres liés au moteur de templates Jinja2.

Exemples de paramètres :

  • extensions : Extensions supplémentaires pour Jinja2.
    extensions = jinja2.ext.do,jinja2.ext.loopcontrols

Priorité et Surcharge des Paramètres

Les paramètres dans ansible.cfg suivent une logique de priorité :

  1. Variables définies en ligne de commande.
  2. Variables dans un fichier ansible.cfg local au projet.
  3. Variables dans ~/.ansible.cfg (configuration utilisateur).
  4. Variables dans /etc/ansible/ansible.cfg (configuration globale).

Bonnes Pratiques pour ansible.cfg

  1. Versionner ansible.cfg : Ajoutez ce fichier dans votre système de contrôle de version pour garantir la cohérence entre les membres de l’équipe.
  2. Centraliser les Paramètres Globaux : Gardez les configurations globales dans /etc/ansible/ansible.cfg et utilisez des fichiers locaux pour les projets spécifiques.
  3. Désactiver les Fichiers .retry : Si vous n’avez pas besoin de reprendre des tâches échouées, désactivez ces fichiers pour éviter le désordre.
  4. Activer le Pipelining : Cela améliore les performances, en particulier sur les connexions SSH.
  5. Utiliser un Fichier de Journalisation : Activez la journalisation pour déboguer les erreurs et surveiller les exécutions.
  6. S'assurer de la Sécurité : Limitez les permissions sur ansible.cfg si des informations sensibles y sont stockées.

Exemple Complet d’un ansible.cfg

[defaults]
inventory = ./inventory
remote_user = ansible
timeout = 30
forks = 10
host_key_checking = False
retry_files_enabled = False
stdout_callback = yaml
log_path = /var/log/ansible.log

[privilege_escalation]
become = True
become_user = root
become_method = sudo
become_ask_pass = False

[ssh_connection]
ssh_args = -o ControlMaster=auto -o ControlPersist=60s
pipelining = True
control_path = %(directory)s/%%h-%%r
scp_if_ssh = True

[galaxy]
server_list = galaxy
role_skeleton = /path/to/skeleton

[logging]
log_path = /var/log/ansible.log
log_level = DEBUG
