1. Droits Requis pour l'Utilisateur ansible

a. Droits d'Accès SSH

  • L'utilisateur ansible doit avoir une paire de clés SSH configurée pour accéder aux nœuds gérés sans mot de passe.
  • Ajoutez la clé publique de ansible au fichier ~/.ssh/authorized_keys sur chaque nœud cible.

b. Privilèges sudo

  • L'utilisateur ansible doit être capable d'exécuter des commandes avec des privilèges administratifs via sudo.
  • Vous pouvez :
    • Autoriser l'utilisateur à exécuter toutes les commandes avec sudo sans mot de passe.
    • Limiter les commandes autorisées pour renforcer la sécurité.

c. Accès Lecture/Écriture

  • L'utilisateur doit avoir les droits suffisants pour lire/écrire dans les répertoires nécessaires, comme :
    • Les fichiers de configuration dans /etc.
    • Les répertoires liés aux applications que vous déployez.

2. Configuration des Droits pour l'Utilisateur ansible

Étape 1 : Créer l'Utilisateur ansible

Créez l'utilisateur ansible sur chaque nœud cible :

sudo adduser ansible

Étape 2 : Configurer l'Accès SSH

  1. Sur le nœud de contrôle, générez une clé SSH si elle n'existe pas déjà :

    ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa_ansible

  2. Déployez la clé publique sur chaque nœud cible :

    ssh-copy-id -i ~/.ssh/id_rsa_ansible ansible@<node_ip>

  3. Assurez-vous que les permissions sont correctes sur les nœuds cibles :

    chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

Étape 3 : Accorder des Privilèges sudo

  1. Ajoutez l'utilisateur ansible au groupe sudo :

    sudo usermod -aG sudo ansible

  2. Configurez sudo pour permettre à l'utilisateur d'exécuter des commandes sans mot de passe. Ajoutez une règle dans /etc/sudoers :

    sudo visudo

    Ajoutez la ligne suivante à la fin du fichier :

    ansible ALL=(ALL) NOPASSWD: ALL

    Remarque : Si vous voulez limiter les commandes autorisées, vous pouvez spécifier une liste précise. Par exemple :

    ansible ALL=(ALL) NOPASSWD: /bin/systemctl, /usr/bin/apt

Étape 4 : Vérifier les Permissions

  1. Connectez-vous en tant que ansible sur un nœud cible :

    ssh ansible@<node_ip>

  2. Testez les privilèges sudo :

    sudo whoami

    Si tout est correctement configuré, la commande doit renvoyer root.

3. Bonnes Pratiques pour l'Utilisateur ansible

a. Restreindre les Permissions

  • Évitez de donner un accès illimité avec sudo si ce n'est pas nécessaire.
  • Utilisez des règles spécifiques dans /etc/sudoers pour limiter les commandes autorisées :
    ansible ALL=(ALL) NOPASSWD: /bin/systemctl restart nginx, /usr/bin/apt-get install

b. Désactiver l'Accès par Mot de Passe

  • Configurez le fichier /etc/ssh/sshd_config pour désactiver l'accès SSH par mot de passe :
    PasswordAuthentication no
    Redémarrez le service SSH :
    sudo systemctl restart sshd

c. Utiliser des Clés SSH Sécurisées

  • Assurez-vous que la clé privée est protégée par une phrase de passe forte.
  • Si vous utilisez Ansible pour automatiser les connexions, vous pouvez configurer un agent SSH (ssh-agent) pour éviter de saisir la phrase de passe à chaque fois.

d. Surveiller les Activités

  • Activez la journalisation pour surveiller les commandes exécutées via sudo par l'utilisateur ansible.
    • Les logs sont généralement disponibles dans /var/log/auth.log sur Ubuntu.

e. Restreindre l'Accès au Réseau

  • Si possible, limitez l'accès SSH de l'utilisateur ansible à certaines IPs ou plages IP en configurant le fichier /etc/ssh/sshd_config :
    Match User ansible
    AllowUsers ansible@192.168.1.0/24

4. Test avec Ansible

  1. Inventaire Exemple :
    Créez un fichier d'inventaire inventory :

    [ubuntu_nodes]
node1 ansible_host=192.168.1.10 ansible_user=ansible
node2 ansible_host=192.168.1.11 ansible_user=ansible

  2. Commande de Test :
    Exécutez une commande Ansible pour vérifier la configuration :

    ansible all -i inventory -m ping

    Si tout est configuré correctement, vous obtiendrez :

    node1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
node2 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}
