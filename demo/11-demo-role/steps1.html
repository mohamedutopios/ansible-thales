Mise à jour du rôle pour déployer un site web statique

Voici les étapes nécessaires pour modifier le rôle afin de déployer le fichier téléchargé et configurer Nginx en conséquence :

1. Configuration de tasks/main.yml

Ajoutez les tâches nécessaires pour télécharger, extraire et déployer les fichiers du site web.

---
# Installer Nginx
- name: Installer Nginx
  ansible.builtin.yum:
    name: nginx
    state: present
  when: ansible_os_family == "RedHat"

- name: Démarrer et activer Nginx
  ansible.builtin.service:
    name: nginx
    state: started
    enabled: true

# Télécharger l'archive du site
- name: Télécharger le site web
  ansible.builtin.get_url:
    url: https://mohamed-formation.s3.eu-west-3.amazonaws.com/html5up-aerial.zip
    dest: /tmp/html5up-aerial.zip
    mode: '0644'

# Extraire l'archive dans le répertoire Nginx
- name: Extraire l'archive du site web
  ansible.builtin.unarchive:
    src: /tmp/html5up-aerial.zip
    dest: /usr/share/nginx/html/
    remote_src: yes

# Configurer Nginx pour servir le site
- name: Déployer le fichier de configuration Nginx
  ansible.builtin.template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
  notify: Redémarrer Nginx

2. Configuration du gestionnaire dans handlers/main.yml

Ajoutez un gestionnaire pour redémarrer Nginx si nécessaire.

---
# Redémarrer Nginx
- name: Redémarrer Nginx
  ansible.builtin.service:
    name: nginx
    state: restarted

3. Modèle Nginx (templates/nginx.conf.j2)

Mettez à jour le fichier de configuration Nginx pour pointer vers le répertoire du site extrait.

user nginx;
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    server {
        listen 80;
        server_name {{ ansible_hostname }};

        location / {
            root /usr/share/nginx/html;
            index index.html;
        }
    }
}

4. Variables par défaut (defaults/main.yml)

Ajoutez des variables pour rendre le rôle plus flexible (par exemple, URL et destination).

---
site_url: https://mohamed-formation.s3.eu-west-3.amazonaws.com/html5up-aerial.zip
nginx_root: /usr/share/nginx/html

5. Exemple de playbook pour utiliser le rôle

Voici un exemple de playbook qui utilise ce rôle :

---
- name: Déployer un site web statique
  hosts: webservers
  become: yes

  roles:
    - webserver

6. Inventaire

Exemple d'inventaire pour cibler les machines distantes :

[webservers]
192.168.1.10 ansible_user=centos ansible_ssh_private_key_file=~/.ssh/id_rsa
192.168.1.11 ansible_user=centos ansible_ssh_private_key_file=~/.ssh/id_rsa

7. Tester le rôle

Pour exécuter le rôle et vérifier que le site est déployé correctement, utilisez la commande suivante :

ansible-playbook -i inventory playbook.yml
