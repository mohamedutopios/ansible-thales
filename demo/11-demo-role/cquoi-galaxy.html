Qu'est-ce qu'Ansible Galaxy ?

Ansible Galaxy est une plateforme communautaire et un outil en ligne de commande qui permet de partager, télécharger, et gérer des rôles et collections Ansible. Cela facilite la réutilisation et le partage de configurations Ansible dans différents projets.

Il se compose de deux aspects principaux :

  1. La plateforme Ansible Galaxy : galaxy.ansible.com

    • Un site web où les développeurs peuvent trouver, télécharger, et partager des rôles ou des collections Ansible créés par d'autres.
    • La plateforme héberge des milliers de rôles et collections pour des cas d'utilisation variés (serveurs web, bases de données, cloud, etc.).

  2. La commande ansible-galaxy :

    • Un outil CLI qui permet de gérer les rôles et collections Ansible localement.
    • Avec cette commande, vous pouvez :
      • Créer des rôles (init).
      • Installer des rôles ou des collections depuis Galaxy ou un autre dépôt (install).
      • Lister les rôles/collections installés (list).
      • Supprimer des rôles/collections (remove).

1. La plateforme Ansible Galaxy

Fonctionnalités de la plateforme

  • Télécharger des rôles ou collections :
    • Vous pouvez rechercher des configurations prêtes à l'emploi pour divers logiciels, infrastructures ou services (par exemple, Nginx, MySQL, Kubernetes, AWS).
  • Partager vos propres rôles :
    • Les utilisateurs peuvent publier leurs rôles/collections pour les rendre disponibles à la communauté.
  • Explorer les rôles populaires :
    • Les rôles sont évalués et classés selon leur qualité et leur popularité.

Accéder à Ansible Galaxy :

Vous pouvez visiter la plateforme ici : https://galaxy.ansible.com.

2. L'outil CLI ansible-galaxy

Principales commandes

Commande Description
ansible-galaxy init <role_name> Crée la structure de base d'un rôle Ansible localement.
ansible-galaxy install <name> Installe un rôle ou une collection depuis Galaxy ou un autre dépôt.
ansible-galaxy collection install <name> Installe une collection depuis Galaxy.
ansible-galaxy list Liste les rôles ou collections installés localement.
ansible-galaxy remove <name> Supprime un rôle ou une collection.
ansible-galaxy collection list Affiche toutes les collections disponibles sur votre machine.

3. Rôles dans Ansible Galaxy

Un rôle dans Ansible Galaxy est une unité réutilisable contenant tout ce qui est nécessaire pour configurer une application ou un service spécifique. Les rôles peuvent inclure :

  • Des tâches (tasks/)
  • Des variables (vars/ et defaults/)
  • Des fichiers statiques (files/)
  • Des modèles (templates/)
  • Des handlers (handlers/)

Exemple de rôle téléchargé depuis Galaxy :

Si vous cherchez un rôle pour installer Nginx, vous pouvez utiliser la commande suivante :

ansible-galaxy install geerlingguy.nginx

Cela installe le rôle de la communauté (par exemple, geerlingguy.nginx) dans le répertoire par défaut des rôles (~/.ansible/roles).

Utilisation du rôle dans un playbook :

Une fois installé, vous pouvez utiliser le rôle comme suit dans un playbook :

---
- name: Installer et configurer Nginx
  hosts: webservers
  roles:
    - geerlingguy.nginx

4. Collections dans Ansible Galaxy

Les collections sont une évolution des rôles. Elles regroupent :

  • Des rôles.
  • Des modules Ansible supplémentaires.
  • Des plugins, comme des filtres ou des modules de connexion.
  • De la documentation.

Télécharger une collection :

Par exemple, pour télécharger la collection community.general, exécutez :

ansible-galaxy collection install community.general

Cela installe la collection dans le répertoire par défaut (~/.ansible/collections).

Utilisation d'un module ou rôle d'une collection :

Une fois installée, vous pouvez utiliser les ressources d'une collection dans un playbook.

Exemple avec un module de la collection community.general :

---
- name: Utilisation d'un module depuis une collection
  hosts: localhost
  tasks:
    - name: Exécuter une commande avec community.general
      community.general.shell:
        cmd: echo "Hello from Ansible Galaxy"

5. Options de configuration pour ansible-galaxy

Vous pouvez personnaliser certains comportements de ansible-galaxy via le fichier ansible.cfg. Par exemple :

Chemin pour les rôles :

[defaults]
roles_path = ./roles:/etc/ansible/roles

Chemin pour les collections :

[defaults]
collections_paths = ./collections:/etc/ansible/collections

6. Travailler sans Internet (environnements isolés)

Si vous travaillez dans un environnement sans Internet, vous pouvez préparer les rôles et collections sur une machine connectée, puis les transférer.

Télécharger des rôles :

Sur une machine connectée, téléchargez un rôle avec :

ansible-galaxy install <role_name> -p ./roles

Archivez ensuite le répertoire roles :

tar -czf roles.tar.gz ./roles

Télécharger des collections :

Pour les collections :

ansible-galaxy collection install <collection_name> -p ./collections

Archivez le répertoire collections :

tar -czf collections.tar.gz ./collections

Transférez ces archives dans votre environnement isolé, décompressez-les, et configurez les chemins dans ansible.cfg.

7. Avantages d'Ansible Galaxy

  • Réutilisation : Pas besoin de recréer des rôles pour des tâches courantes (ex. : installation de Nginx, gestion d'AWS).
  • Communauté : Accès à des configurations éprouvées créées par des experts.
  • Standardisation : Les rôles téléchargés suivent une structure standardisée.
  • Gain de temps : Installation rapide de rôles et de collections préconçus.
