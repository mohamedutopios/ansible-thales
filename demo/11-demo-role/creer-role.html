1. Commande pour créer un rôle

La manière la plus simple de créer un rôle est d'utiliser la commande suivante :

ansible-galaxy init <role_name>

Exemple :

Pour créer un rôle nommé webserver :

ansible-galaxy init webserver

Cette commande génère automatiquement une structure de répertoire standard pour le rôle.

2. Structure générée d'un rôle

Une fois le rôle créé, vous obtenez une structure comme ceci :

webserver/
├── defaults/          # Variables par défaut
│   └── main.yml
├── files/             # Fichiers statiques à copier sur les hôtes
├── handlers/          # Handlers pour les actions conditionnées (ex: restart service)
│   └── main.yml
├── meta/              # Métadonnées du rôle (dépendances, informations)
│   └── main.yml
├── tasks/             # Tâches principales à exécuter
│   └── main.yml
├── templates/         # Modèles Jinja2 à utiliser pour les fichiers de configuration
├── tests/             # Tests pour le rôle
│   ├── inventory
│   └── test.yml
└── vars/              # Variables spécifiques au rôle
    └── main.yml

3. Description de chaque répertoire

Répertoire Utilisation
defaults Contient les variables par défaut du rôle. Ces variables ont la priorité la plus basse.
files Contient les fichiers statiques à copier sur les hôtes distants.
handlers Définit des actions déclenchées par des notifications, comme redémarrer un service.
meta Contient des métadonnées sur le rôle, comme ses dépendances ou l'auteur.
tasks Liste des tâches principales à exécuter pour ce rôle.
templates Contient les fichiers modèles Jinja2 pour générer des fichiers dynamiques.
tests Fournit un playbook et un inventaire de test pour valider le rôle.
vars Définit des variables spécifiques au rôle. Ces variables ont une priorité plus élevée que celles dans defaults.

4. Remplir les fichiers du rôle

a. Ajouter des tâches dans tasks/main.yml

Les tâches principales du rôle se trouvent dans ce fichier. Exemple : Installer et configurer Nginx :

---
# Installer Nginx
- name: Installer le paquet Nginx
  ansible.builtin.yum:
    name: nginx
    state: present
  when: ansible_os_family == "RedHat"

# Démarrer et activer le service Nginx
- name: Activer et démarrer Nginx
  ansible.builtin.service:
    name: nginx
    state: started
    enabled: true

b. Ajouter des variables par défaut dans defaults/main.yml

Les variables par défaut utilisées dans le rôle. Exemple :

---
nginx_port: 80

c. Ajouter un handler dans handlers/main.yml

Les handlers exécutent des actions lorsque des tâches les notifient. Exemple : Redémarrer Nginx si la configuration change :

---
- name: Redémarrer Nginx
  ansible.builtin.service:
    name: nginx
    state: restarted

d. Ajouter un modèle dans templates/nginx.conf.j2

Un modèle Jinja2 pour la configuration Nginx. Exemple :

server {
    listen {{ nginx_port }};
    server_name {{ ansible_hostname }};

    location / {
        root /usr/share/nginx/html;
        index index.html;
    }
}

e. Définir les métadonnées dans meta/main.yml

Définissez les informations sur le rôle et ses dépendances éventuelles :

---
galaxy_info:
  author: Your Name
  description: Configuration de Nginx comme serveur web.
  company: Your Company
  license: MIT
dependencies: []

5. Utiliser le rôle dans un playbook

Une fois que le rôle est prêt, vous pouvez l'utiliser dans un playbook.

Exemple de playbook :

---
- name: Déployer un serveur web
  hosts: webservers
  become: yes

  roles:
    - webserver

Exemple d'inventaire :

[webservers]
192.168.1.10 ansible_user=centos ansible_ssh_private_key_file=~/.ssh/id_rsa
192.168.1.11 ansible_user=centos ansible_ssh_private_key_file=~/.ssh/id_rsa

Exécutez le playbook avec la commande :

ansible-playbook -i inventory playbook.yml

6. Ajouter le rôle à un projet existant

Pour utiliser votre rôle dans un projet, placez-le dans un répertoire dédié, généralement appelé roles/.

Exemple de structure du projet :

project/
├── ansible.cfg
├── inventory
├── playbook.yml
└── roles/
    └── webserver/
        ├── defaults/
        ├── files/
        ├── handlers/
        ├── meta/
        ├── tasks/
        ├── templates/
        └── vars/

Dans ansible.cfg, configurez le chemin vers les rôles :

[defaults]
roles_path = ./roles:/etc/ansible/roles

7. Tester le rôle

Pour valider que votre rôle fonctionne comme prévu, testez-le en local ou dans un environnement de test.

a. Créer un fichier de test

Dans le répertoire tests/ du rôle, utilisez test.yml pour tester uniquement ce rôle :

---
- name: Tester le rôle webserver
  hosts: localhost
  roles:
    - webserver

b. Exécuter le test

ansible-playbook -i localhost, tests/test.yml

c. Simulation (dry-run)

Exécutez en mode simulation pour voir ce qu'Ansible ferait sans appliquer les modifications :

ansible-playbook -i inventory playbook.yml --check
