1. Création de la structure du rôle

Vous pouvez créer la structure de base d'un rôle en utilisant la commande :

ansible-galaxy init webserver

Cela génère la structure suivante :

webserver/
├── defaults/          # Variables par défaut
│   └── main.yml
├── files/             # Fichiers statiques à copier sur l'hôte
├── handlers/          # Actions déclenchées par des notifications
│   └── main.yml
├── meta/              # Métadonnées du rôle
│   └── main.yml
├── tasks/             # Liste des tâches à exécuter
│   └── main.yml
├── templates/         # Fichiers modèles Jinja2
├── tests/             # Tests pour le rôle
│   ├── inventory
│   └── test.yml
└── vars/              # Variables spécifiques au rôle
    └── main.yml

2. Exemple de rôle "webserver"

a. Configuration de tasks/main.yml

Le fichier tasks/main.yml contient les tâches principales du rôle, par exemple, installer et configurer Nginx :

---
# Installer le serveur web Nginx
- name: Installer Nginx
  ansible.builtin.yum:
    name: nginx
    state: present
  when: ansible_os_family == "RedHat"

- name: Démarrer et activer Nginx
  ansible.builtin.service:
    name: nginx
    state: started
    enabled: true

- name: Déployer le fichier de configuration Nginx
  ansible.builtin.template:
    src: nginx.conf.j2
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
  notify: Redémarrer Nginx

b. Configuration de handlers/main.yml

Les handlers sont utilisés pour exécuter des actions (comme redémarrer un service) uniquement si elles sont notifiées par une tâche :

---
# Redémarrer Nginx si la configuration change
- name: Redémarrer Nginx
  ansible.builtin.service:
    name: nginx
    state: restarted

c. Fichier de modèle (templates/nginx.conf.j2)

Le fichier de configuration nginx.conf.j2 est un modèle Jinja2. Voici un exemple de configuration simple pour Nginx :

user nginx;
worker_processes auto;

events {
    worker_connections 1024;
}

http {
    server {
        listen 80;
        server_name {{ ansible_hostname }};

        location / {
            root /usr/share/nginx/html;
            index index.html;
        }
    }
}

d. Variables par défaut (defaults/main.yml)

Les variables par défaut sont définies ici. Elles peuvent être surchargées par des variables définies dans les playbooks :

---
nginx_port: 80

3. Utilisation du rôle dans un playbook

Une fois le rôle créé, vous pouvez l'utiliser dans un playbook pour appliquer sa configuration.

Exemple de playbook :

---
- name: Configuration d'un serveur web
  hosts: webservers
  become: yes

  roles:
    - webserver

Exemple d'inventaire :

[webservers]
192.168.1.10 ansible_user=centos ansible_ssh_private_key_file=~/.ssh/id_rsa
192.168.1.11 ansible_user=centos ansible_ssh_private_key_file=~/.ssh/id_rsa

4. Tester le rôle

Pour tester que le rôle fonctionne correctement, vous pouvez exécuter le playbook :

ansible-playbook -i inventory playbook.yml

5. Étendre le rôle

Vous pouvez enrichir le rôle en :

  • Ajoutant des fichiers statiques à copier dans le répertoire files/.
  • Gérant des différents systèmes d'exploitation avec des conditions when.
  • Utilisant des variables spécifiques définies dans vars/main.yml pour personnaliser davantage la configuration.
