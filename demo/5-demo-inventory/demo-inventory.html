Inventaire

Voici un exemple de fichier inventory.ini basé sur votre structure :

[ansible]
ansible ansible_host=192.168.60.10 ansible_port=22 ansible_user=root ansible_private_key_file=~/.ssh/id_rsa

[nodes]
node1 ansible_host=192.168.60.11 ansible_port=22 ansible_user=root ansible_private_key_file=~/.ssh/id_rsa
node2 ansible_host=192.168.60.12 ansible_port=22 ansible_user=root ansible_private_key_file=~/.ssh/id_rsa
node3 ansible_host=192.168.60.13 ansible_port=22 ansible_user=root ansible_private_key_file=~/.ssh/id_rsa

[web]
node1
node2

[db]
node3

[all_nodes:children]
web
db

[web:vars]
ansible_user=root
ansible_port=22
ansible_private_key_file=~/.ssh/id_rsa

[db:vars]
ansible_user=root
ansible_port=22
ansible_private_key_file=~/.ssh/id_rsa

1. Tester la Connectivité de l'Inventaire

Commande :

ansible all -i inventory.ini -m ping

Résultat attendu :

node1 | SUCCESS => {"changed": false, "ping": "pong"}
node2 | SUCCESS => {"changed": false, "ping": "pong"}
node3 | SUCCESS => {"changed": false, "ping": "pong"}
ansible | SUCCESS => {"changed": false, "ping": "pong"}

2. Exécuter une Commande sur Tous les Nœuds

Commande :

ansible all_nodes -i inventory.ini -m shell -a "uptime"

Résultat attendu :

Chaque nœud retourne son temps d'activité.

3. Tester les Groupes

a. Tester le Groupe web

ansible web -i inventory.ini -m ping

b. Tester le Groupe db

ansible db -i inventory.ini -m ping

4. Installer un Logiciel par Groupe

Installez nginx sur les nœuds du groupe web.

Commande :

ansible web -i inventory.ini -m apt -a "name=nginx state=present update_cache=true" --become

Résultat attendu :

nginx sera installé sur node1 et node2.

5. Déployer un Fichier sur Tous les Nœuds

Copiez un fichier /etc/hosts local vers tous les nœuds dans /tmp/hosts_backup.

Commande :

ansible all_nodes -i inventory.ini -m copy -a "src=/etc/hosts dest=/tmp/hosts_backup"

6. Redémarrer un Service sur un Groupe

Redémarrez le service SSH sur les nœuds db.

Commande :

ansible db -i inventory.ini -m service -a "name=ssh state=restarted"

7. Exécuter un Playbook sur Tous les Nœuds

Créez un Playbook pour installer nginx sur le groupe web et mysql sur le groupe db.

Fichier playbook.yml :

---
- name: Configure Web Servers
  hosts: web
  tasks:
    - name: Install Nginx
      apt:
        name: nginx
        state: present
        update_cache: true
      become: true

- name: Configure Database Server
  hosts: db
  tasks:
    - name: Install MySQL
      apt:
        name: mysql-server
        state: present
        update_cache: true
      become: true

Commande pour exécuter le Playbook :

ansible-playbook -i inventory.ini playbook.yml

Résultat attendu :

  • nginx sera installé sur les nœuds du groupe web (node1, node2).
  • mysql-server sera installé sur le nœud du groupe db (node3).

8. Filtrer les Commandes avec des Tags

Ajoutez des tags à un Playbook pour exécuter uniquement certaines parties.

Playbook avec Tags (playbook_with_tags.yml) :

---
- name: Configure Web Servers
  hosts: web
  tasks:
    - name: Install Nginx
      apt:
        name: nginx
        state: present
        update_cache: true
      become: true
      tags:
        - nginx

- name: Configure Database Server
  hosts: db
  tasks:
    - name: Install MySQL
      apt:
        name: mysql-server
        state: present
        update_cache: true
      become: true
      tags:
        - mysql

Exécuter uniquement les tâches liées à nginx :

ansible-playbook -i inventory.ini playbook_with_tags.yml --tags nginx

Exécuter uniquement les tâches liées à mysql :

ansible-playbook -i inventory.ini playbook_with_tags.yml --tags mysql

9. Exécuter une Commande sur un Seul Nœud

Exécutez une commande sur un nœud spécifique (par exemple, node1).

Commande :

ansible node1 -i inventory.ini -m shell -a "hostname"

Résultat attendu :

node1 | SUCCESS => {
    "changed": true,
    "stdout": "node1",
    "stderr": "",
    "rc": 0
}

10. Afficher des Informations sur Tous les Nœuds

Utilisez le module setup pour collecter des faits sur tous les nœuds.

Commande :

ansible all_nodes -i inventory.ini -m setup

Résultat attendu :

Renvoie des informations détaillées (facter) pour chaque nœud, comme le système d'exploitation, la mémoire, le CPU, etc.
