Objectif :

Créer un fichier /etc/hosts centralisé pour tous les nodes en regroupant leurs adresses IP et noms d'hôtes.

Template Jinja (templates/centralized_hosts.j2) :

# Fichier /etc/hosts généré par Ansible

127.0.0.1   localhost

{% for host in groups['all'] %}
{{ hostvars[host]['ansible_host'] }}   {{ host }}
{% endfor %}

Exemple d'inventaire (inventory/hosts.yml) :

all:
  children:
    nodes:
      hosts:
        node4:
          ansible_host: 192.168.60.14
        node5:
          ansible_host: 192.168.60.15
        node6:
          ansible_host: 192.168.60.16
    ansible:
      hosts:
        ansible-inv:
          ansible_host: 192.168.60.10

Playbook Ansible (playbooks/generate_hosts.yaml) :

Voici le playbook qui utilise le template Jinja pour générer le fichier /etc/hosts.

- name: Générer un fichier centralisé /etc/hosts
  hosts: localhost
  tasks:
    - name: Générer le fichier /etc/hosts à partir du template
      template:
        src: templates/centralized_hosts.j2
        dest: /tmp/centralized_hosts
      vars:
        groups: "{{ groups }}"
        hostvars: "{{ hostvars }}"

    - name: Afficher le contenu du fichier généré
      debug:
        msg: "{{ lookup('file', '/tmp/centralized_hosts') }}"

Résultat attendu :

Lorsque vous exécutez le playbook, il génère un fichier /tmp/centralized_hosts contenant les informations des nodes. Exemple :

# Fichier /etc/hosts généré par Ansible

127.0.0.1   localhost

192.168.60.14   node4
192.168.60.15   node5
192.168.60.16   node6
192.168.60.10   ansible-inv

Comment exécuter le Playbook ?

Exécutez la commande suivante pour générer le fichier :

ansible-playbook -i inventory/hosts.yml playbooks/generate_hosts.yaml

Explications :

  1. Template Jinja :

    • La boucle for parcourt tous les hôtes définis dans le groupe all.
    • hostvars[host]['ansible_host'] récupère l'adresse IP de chaque node.
    • Le nom d'hôte est directement récupéré depuis la clé host.

  2. Fichier généré :

    • Le fichier /tmp/centralized_hosts regroupe toutes les adresses IP et noms des nodes.
