1. Tester la connectivité entre Ansible et les nœuds

Commande ping :

Permet de vérifier si Ansible peut communiquer avec les hôtes dans l’inventaire :

ansible -i inventory nodes -m ping
  • -i inventory : Spécifie le fichier d’inventaire.
  • -m ping : Utilise le module ping d'Ansible.

Exemple de sortie :

node1 | SUCCESS => {
    "changed": false,
    "ping": "pong"
}

2. Exécuter des commandes sur les nœuds

Exécution d’une commande simple :

Utilise le module command pour exécuter une commande shell (sans interpréteur comme Bash) sur les nœuds.

ansible -i inventory nodes -m command -a "uptime"

Exécution d’une commande avec interpréteur :

Utilise le module shell pour exécuter des commandes nécessitant un interpréteur (comme des redirections ou des variables d’environnement).

ansible -i inventory nodes -m shell -a "echo $HOME"

3. Gestion des fichiers

Créer un fichier :

Utilise le module file pour créer un fichier ou un répertoire.

ansible -i inventory nodes -m file -a "path=/tmp/test-file state=touch"

Supprimer un fichier ou un répertoire :

Supprime un fichier ou un répertoire spécifié.

ansible -i inventory nodes -m file -a "path=/tmp/test-file state=absent"

Copier un fichier depuis Ansible vers les nœuds :

Utilise le module copy pour transférer des fichiers :

ansible -i inventory nodes -m copy -a "src=/home/inventory dest=/tmp/remote-file"

Télécharger un fichier des nœuds vers Ansible :

Utilise le module fetch pour récupérer un fichier distant.

ansible -i inventory nodes -m fetch -a "src=/tmp/remote-file dest=/path/to/local/directory"

4. Gestion des paquets

Installer un paquet :

Utilise le module apt (pour les distributions basées sur Debian comme Ubuntu) pour installer un paquet :

ansible -i inventory nodes -m apt -a "name=htop state=present"

Supprimer un paquet :

ansible -i inventory nodes -m apt -a "name=htop state=absent"

5. Gestion des services

Démarrer un service :

Utilise le module service pour démarrer un service :

ansible -i inventory nodes -m service -a "name=ssh state=started"

Arrêter un service :

ansible -i inventory nodes -m service -a "name=ssh state=stopped"

Redémarrer un service :

ansible -i inventory nodes -m service -a "name=ssh state=restarted"

6. Gestion des utilisateurs

Créer un utilisateur :

Utilise le module user pour ajouter un utilisateur.

ansible -i inventory nodes -m user -a "name=newuser state=present"

Supprimer un utilisateur :

ansible -i inventory nodes -m user -a "name=newuser state=absent"

7. Gestion des fichiers et permissions

Changer les permissions d’un fichier :

Utilise le module file pour gérer les permissions et le propriétaire d’un fichier :

ansible -i inventory nodes -m file -a "path=/tmp/test-file mode=0644 owner=root group=root"

8. Exécuter un playbook

Pour automatiser des tâches plus complexes, exécutez un playbook YAML :

ansible-playbook -i inventory setup.yml
  • setup.yml est le fichier contenant vos tâches Ansible.

9. Collecte d’informations système

Obtenir des informations sur le système :

Utilise le module setup pour collecter des faits sur les nœuds.

ansible -i inventory nodes -m setup

Filtrer les informations collectées :

Par exemple, pour afficher uniquement l'adresse IP :

ansible -i inventory nodes -m setup -a "filter=ansible_default_ipv4"

10. Gestion de l’inventaire dynamique

Lister les hôtes de l’inventaire :

Affiche les hôtes configurés dans l’inventaire.

ansible-inventory -i inventory --list

Tester une connexion avec un inventaire dynamique :

ansible -i inventory all -m ping

11. Déboguer les connexions SSH

Utilisez le mode debug pour diagnostiquer les problèmes de connectivité :

ansible -i inventory nodes -m ping -vvv

12. Gestion de la configuration réseau

Changer l’adresse IP :

Utilise le module command pour modifier l’adresse IP (exemple avec ifconfig) :

ansible -i inventory nodes -m command -a "ifconfig eth0 192.168.1.10 netmask 255.255.255.0"
