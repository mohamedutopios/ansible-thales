Étape 1 : Créer galaxy.yml

Le fichier galaxy.yml est obligatoire pour définir la collection et ses métadonnées.

namespace: my_namespace
name: my_collection
version: 1.0.0
readme: README.md
authors:
  - "Votre Nom <votre.email@example.com>"
description: "Une collection Ansible pour installer et configurer Apache."
license: MIT
dependencies: {}

Étape 2 : Ajouter un rôle Apache

1. roles/apache/tasks/main.yml

La tâche principale appelle d'autres fichiers pour structurer les étapes.

- name: Inclure les tâches d'installation
  include_tasks: install.yml

2. roles/apache/tasks/install.yml

Ce fichier contient les étapes pour installer Apache.

- name: Installer Apache
  ansible.builtin.yum:
    name: httpd
    state: present

- name: Copier le fichier de configuration Apache
  ansible.builtin.template:
    src: apache.conf.j2
    dest: /etc/httpd/conf/httpd.conf
    mode: 0644

- name: Copier la page d'accueil
  ansible.builtin.copy:
    src: index.html
    dest: /var/www/html/index.html
    mode: 0644

- name: Démarrer et activer Apache
  ansible.builtin.service:
    name: httpd
    state: started
    enabled: yes

3. roles/apache/handlers/main.yml

Définir un gestionnaire pour redémarrer Apache si nécessaire.

- name: Redémarrer Apache
  ansible.builtin.service:
    name: httpd
    state: restarted

4. roles/apache/defaults/main.yml

Les variables par défaut pour le rôle.

apache_port: 80

5. roles/apache/vars/main.yml

Des variables spécifiques au rôle.

apache_user: apache
apache_group: apache

6. roles/apache/templates/apache.conf.j2

Un fichier de configuration Apache.

<VirtualHost *:{{ apache_port }}>
    DocumentRoot "/var/www/html"
    ErrorLog "/var/log/httpd/error.log"
    CustomLog "/var/log/httpd/access.log" common
</VirtualHost>

7. roles/apache/files/index.html

Une page HTML simple.

<!DOCTYPE html>
<html>
<head>
    <title>Bienvenue</title>
</head>
<body>
    <h1>Bienvenue sur Apache installé avec Ansible</h1>
</body>
</html>

Étape 3 : Ajouter un module personnalisé

1. plugins/modules/my_custom_module.py

Un module Python simple qui affiche un message.

#!/usr/bin/python
from ansible.module_utils.basic import AnsibleModule

def main():
    module = AnsibleModule(argument_spec={})
    module.exit_json(changed=False, message="Bonjour depuis un module personnalisé !")

if __name__ == '__main__':
    main()

Étape 4 : Ajouter des tests

1. tests/inventory

Un inventaire pour tester la collection.

[all]
localhost ansible_connection=local

2. tests/test.yml

Un playbook pour tester la collection.

- name: Tester la collection Apache
  hosts: all
  roles:
    - my_namespace.my_collection.apache

Étape 5 : Utiliser et tester la collection

1. Créer la collection localement

Exécutez la commande suivante pour packager la collection :

ansible-galaxy collection build

Cela génère un fichier .tar.gz. Installez-le localement avec :

ansible-galaxy collection install my_namespace-my_collection-1.0.0.tar.gz

2. Tester avec le playbook

Exécutez le playbook de test :

ansible-playbook -i tests/inventory tests/test.yml

Étape 6 : Publier la collection

  1. Se connecter à Ansible Galaxy :
    • Configurez votre compte Galaxy.
    • Publiez la collection :
      ansible-galaxy collection publish my_namespace-my_collection-1.0.0.tar.gz
