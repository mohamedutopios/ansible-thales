1. Pré-requis

  1. Installer Ansible Galaxy et Ansible (si ce n’est pas déjà fait) :

    sudo apt update
sudo apt install ansible -y

  2. Créer un compte sur Ansible Galaxy si tu n'en as pas encore.
    Connecte-toi avec ton compte GitHub.

2. Créer la Collection

  1. Initialise la collection avec la commande :

    Remplace your_namespace par ton nom d’utilisateur GitHub (c'est important pour la publication).

    ansible-galaxy collection init your_namespace.fortigate_demo

  2. Structure créée automatiquement :

    your_namespace/
└── fortigate_demo/
    ├── galaxy.yml
    ├── README.md
    ├── roles/
    │   └── configure_interface/
    │       ├── tasks/
    │       ├── vars/
    │       ├── templates/
    │       ├── handlers/
    │       └── meta/
    └── playbooks/

3. Développer les Rôles

Utilise l'exemple de la collection Fortigate que nous avons vu précédemment pour remplir les fichiers tasks/main.yml, vars/main.yml, etc.

4. Modifier le Fichier galaxy.yml

Le fichier galaxy.yml est crucial pour la publication. Voici un exemple :

namespace: your_namespace
name: fortigate_demo
version: 1.0.0
readme: README.md
authors:
  - Your Name <your_email@example.com>
description: A collection to configure Fortigate interfaces and firewall policies.
license: MIT
tags:
  - fortigate
  - firewall
  - network
dependencies: {}
repository: https://github.com/your_namespace/fortigate_demo

5. Préparer le Dépôt GitHub

  1. Initialise un dépôt Git dans ton projet :

    cd your_namespace.fortigate_demo
git init
git add .
git commit -m "Initial commit for Fortigate Ansible Collection"

  2. Crée un dépôt sur GitHub (via l'interface web ou en ligne de commande).

  3. Ajoute le dépôt distant et pousse ton code :

    git remote add origin https://github.com/your_namespace/fortigate_demo.git
git branch -M main
git push -u origin main

6. Packager la Collection

Utilise la commande suivante pour créer un package .tar.gz de ta collection :

ansible-galaxy collection build

Cela générera un fichier similaire à :

your_namespace-fortigate_demo-1.0.0.tar.gz

7. Publier la Collection sur Ansible Galaxy

  1. Récupérer ton API Token sur Ansible Galaxy :

    • Connecte-toi sur Ansible Galaxy.
    • Va dans My Content → API Token et copie le token.

  2. Publier la collection :

    Remplace <your_token> par ton token API et le nom du fichier .tar.gz par celui généré.

    ansible-galaxy collection publish your_namespace-fortigate_demo-1.0.0.tar.gz --api-key <your_token>

8. Vérification sur Ansible Galaxy

  • Une fois publiée, ta collection sera visible sur Ansible Galaxy.

9. Utilisation de la Collection

  1. Installer la collection depuis Ansible Galaxy :

    ansible-galaxy collection install your_namespace.fortigate_demo

  2. Utiliser dans un playbook :

    - name: Use Fortigate Collection
  hosts: fortigates
  gather_facts: no
  roles:
    - your_namespace.fortigate_demo.configure_interface
    - your_namespace.fortigate_demo.configure_firewall

Récapitulatif des Commandes Clés :

  1. Initialiser la collection :
    ansible-galaxy collection init your_namespace.fortigate_demo

  2. Packager la collection :
    ansible-galaxy collection build

  3. Publier sur Ansible Galaxy :
    ansible-galaxy collection publish your_namespace-fortigate_demo-1.0.0.tar.gz --api-key <your_token>
